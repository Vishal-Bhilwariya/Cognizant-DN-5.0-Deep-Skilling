## 📅 Week 1 — Branch: `week_1`

### Module 1 – Design Patterns and Principles

#### 1) OOPs (Object-Oriented Programming)
- a) History of OOPs
  - Evolution: Machine Language → Assembly → Procedural → OOP
  - Why OOP was needed for large-scale applications
- b) Object & Class
  - Class as a blueprint, Object as an instance
  - Fields, Methods, Constructors (Default, Parameterized, Copy)
- c) OOP vs Procedural Programming
  - Key differences in approach, security, reusability, and maintainability
- d) Abstraction
  - Hide "How", Show "What"
  - Abstract classes and interfaces
- e) Encapsulation
  - Wrapping data + methods into a single class
  - Private variables with Getters & Setters, data validation
- f) Inheritance
  - Parent-Child class relationship using `extends`
  - Types: Single, Multilevel, Hierarchical
- g) Polymorphism
  - Compile-time (Method Overloading)
  - Runtime (Method Overriding)
- h) Access Modifiers
  - `private`, `default`, `protected`, `public`
  - Access level comparison table
- i) Difference Between the Four Pillars of OOP
  - Comparison of Encapsulation, Inheritance, Polymorphism, Abstraction

#### 2) SOLID Principles
- a) UML Diagrams
- b) Introduction to SOLID

#### Interview Questions
- Design Patterns & Principles related Q&A
